Portal, not picker.// a reframing

The one-pager says "pick a harness like picking a car." For a 60-year-old grandma in Europe or a 50-year-old dad in Delhi, that's already too much choice. This is how I'd soften it without losing the strategic moat.

A non-technical user doesn't pick between Claude Code and Codex — they don't even know what those words mean. A harness picker as the first screen asks them to make a technical decision before they've gotten any value.

06 / SCOPEA short list of what we build — and a longer list of what we don't.

The scope discipline is what keeps the portal thin. Every "what we don't build" is a reason we can swap harnesses.

What we build

  • Portal UI — chat surface, harness picker (hidden), artifacts
  • Resource provisioning — computer, phone, email, card
  • Skills / plugins layer that works with any harness
  • Orb Browser integration — live feed via URL
  • OrbCloud API integration
  • Billing, auth, user management

What we don't build

  • LLM orchestration / agent loops
  • Tool execution engines
  • Model routing or provider management
  • Context window management
  • Code editing / file system intelligence
  • Any AI/ML logic whatsoever

07 / DECIDE BEFORE WE DESIGNThree questions I need answered.

The portal UX changes shape depending on these. I don't want to guess — a wrong call on any of them means weeks of wasted design.

Q 01

Doing things, or making things?

SPOQ 1.0 leaned maker (code, docs, sites). The PDF leans doer (book, shop, plan). A code-writing portal and a life-admin portal are almost completely different products.

Doer Maker Both
Q 02

Hidden harness, or pro-mode toggle?

My instinct is hide entirely for v1 — one default, no picker. But if your earliest users are devs from Twitter/HN, a pro-mode toggle might matter from day one for distribution.

Hidden Pro toggle Picker up-front
Q 03

Fresh repo, or reuse spoq-website?

The PDF says "1 repo, thin portal frontend." We already have spoq-website, spoq-conductor, spoq-ios, spoq-web-apis. Worth being explicit about what dies.

Fresh Reuse site Fold all in
