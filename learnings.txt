2026-04-22 — Correction from Sam: don't list keys as prereqs.
The plan was always dynamic/JIT — the agent asks for a capability only when
the task hits it, then the portal surfaces an inline "connect X?" prompt.
Reframed: we don't need ANY external creds to keep building. Build the
capability REQUEST PROTOCOL with stub handlers; real services wire up per
capability later, only when a canonical task actually trips them.

Shipped the JIT protocol:
  - System preamble (buildSystemPreamble) teaches the agent to emit
    <<SPOQ-NEED type="identity|email.send|phone.sms|card.charge" reason="..."/>>
    only when the task genuinely needs the capability.
  - Sandbox run() now accepts appendSystemPrompt → wired via claude-code's
    --append-system-prompt flag.
  - /api/chat parses SSE stdout, strips tokens from user-visible text, emits
    { kind: "capability_request", request } events.
  - JITResource component renders inline "Connect / Skip" prompt.
  - On connect, client stubs a label (stubConnect) and re-sends the full
    conversation with caps state set so the agent sees the capability listed
    under "Already connected:" in the preamble.
  - verify_jit.py: 3 scenarios (positive, negative, resume) all PASS end-to-end.

Key insight: the canonical suite's c05 (write thank-you email) correctly
does NOT trip email.send — it's pure drafting. The "send email to Priya"
task DOES trip it. The preamble's drafting-vs-sending distinction works.

2026-04-22 — Capability catalog opened up. No more hardcoded 4-item menu
(identity / email.send / phone.sms / card.charge). The agent now emits
<<SPOQ-NEED type="<composio-toolkit-slug>" .../>> for any toolkit (gmail,

<Prettified Slug>?" — label derived on the fly, no lookup table.

2026-04-24 — UX pass: proactive greeting + anti-AI voice + simpler protocol.
Sam's direction: "this project wins on practicality, not technicality."
Several radical simplifications landed in one session.

  - Landing and /chat collapsed. / renders ChatSession directly.
    /chat route + ChatBox.tsx deleted. No URL hop, no serif headline,
    no suggestion chips. Conversation from pixel one.

  - Agent greets first, client-side only. On mount with no prefilled q,
    a hardcoded greeting types out char-by-char at ~55cps. Zero GLM
    cost per visitor, feels alive, survives scroll and user input
    (greeting only ever mutates items[0]).

  - Anti-AI voice rules baked into BOTH CLAUDE.md and preamble.ts.
    Source: null-bytes/grant-drafter/knowledge/writing-style/anti-ai-
    writing-guide.md + Sam feedback memories. Banned vocab (delve,
    landscape, harness, leverage, utilize, robust, etc), no em dashes,
    no rhetorical closers, sentences >= 5 words, no sycophantic
    openers, use contractions.

  - JIT protocol simplified. Old preamble: emit tag AND write "I'll
    need your Gmail — is that ok?". Doublespeak since the card
    already asks. New rule: emit one tag and stop. On connect, a
    hidden user-role nudge ("Connected: gmail as priya@... Continue
    the task without commentary.") kicks the agent back into work
    without showing as a user bubble. Skip path is symmetric.

  - Reload gap closed. user_id persists via localStorage, Composio
    holds the OAuth tokens, but CapabilityState was React-only so a
    refresh forgot connections until the agent hit the capability
    again. New GET /api/capabilities/seed?user_id=... queries
    composio.connectedAccounts.list and rebuilds state on mount.
    Graceful when no key / no user (returns {caps:{}}).

  - Resume-nudge voice. User message items got an optional `hidden`
    flag — included in the API conversation but skipped in render.
    The italic "connected gmail" chip from JITResource is the only
    visible feedback.
